02 Four paradigm shifts

Scale is a training recipe

Bigger models on more diverse data with better data curation — not novel architectures — drove most 2024–2025 capability jumps. Data quality > data quantity.

Post-training > pretraining (ROI)

RLHF, DPO, GRPO. The same base model with different post-training produces vastly different capabilities. RL from verifiable rewards unlocks reasoning without labeled CoT.

Modalities converge

Text, image, video, audio — one model, shared transformer backbone, modality-specific encoders. Contrastive alignment + cross-attention fusion is the dominant recipe.

Inference is a research problem

Speculative decoding, INT4 quant, continuous batching, paged KV. Test-time compute (more tokens = better answers) means inference cost is now a primary design constraint.

03 RMSNorm & SwiGLU training

Two architectural micro-improvements that are now standard in every major LLM. Both improve training stability with less compute.

RMSNorm — drop the mean, keep the scale

LayerNorm vs RMSNorm LayerNorm: ȳ = (x − μ) / √(σ² + ε) · γ + β   ← needs mean + variance
RMSNorm: ȳ = x / RMS(x) · γ                 ← only RMS: √(mean(x²))

No mean subtraction. No β offset. ~10–20% faster. Same training stability. 
pytorch
# RMSNorm — native in PyTorch 2.4+
norm = nn.RMSNorm(d_model)

# Manual implementation
class RMSNorm(nn.Module):
    def __init__(self, d, eps=1e-8):
        super().__init__()
        self.gamma = nn.Parameter(torch.ones(d))
        self.eps   = eps

    def forward(self, x):
        rms = x.pow(2).mean(-1, keepdim=True).add(self.eps).sqrt()
        return self.gamma * (x / rms)

SwiGLU — gated activation for FFN

SwiGLU — used in LLaMA, Qwen3, Mistral, GPT-4 SwiGLU(x) = SiLU(W₁x) ⊙ W₂x → W₃
SiLU(x) = x · σ(x)   ← "swish" — smooth, non-monotonic

Three weight matrices instead of two. Hidden dim = (2/3)·4d to keep FLOP count equal.
Consistently improves perplexity ~0.5–1 point vs ReLU FFN. All frontier models use it. 
pytorch — SwiGLU FFN
class SwiGLUFFN(nn.Module):
    def __init__(self, d_model, expand=4):
        super().__init__()
        hidden = int(d_model * expand * 2 / 3)  # iso-FLOP to standard 4× FFN
        self.gate = nn.Linear(d_model, hidden, bias=False)
        self.up   = nn.Linear(d_model, hidden, bias=False)
        self.down = nn.Linear(hidden,  d_model, bias=False)

    def forward(self, x):
        return self.down(F.silu(self.gate(x)) * self.up(x))

04 RoPE & long context LLM

Rotary Position Embedding encodes relative position by rotating Q and K vectors in complex space before the attention dot product. It's the dominant position encoding in all 2024–2025 models (LLaMA-3, Qwen3, Mistral, Gemma).

RoPE — key property qₘᵀkₙ = Re[(Rₘq)ᴴ(Rₙk)] = f(q, k, m−n)
← attention score depends only on (m−n), not absolute positions m and n

Rₘ = block-diagonal rotation matrix with frequency θᵢ = 10000^(−2i/d)
Applying Rₘ to a vector rotates each 2D subspace by angle m·θᵢ

Extending context beyond training length

YaRN

Yet Another RoPE extensioN. Interpolates + rescales RoPE frequencies to extend context. LLaMA-3 uses YaRN to go from 8k to 128k. Add a short fine-tuning stage on long documents.

LongRoPE

Non-uniformly rescales different frequency groups. Low-frequency dimensions (long range) scaled less than high-frequency (short range). Better quality at very long contexts.

Ring Attention

Distributes the attention computation for a single sequence across multiple GPUs in a ring topology. Enables sequences of 1M+ tokens. Used for Gemini's 2M context.

Sliding Window (Mistral)

Each token attends only to a local window (e.g. 4096 tokens) plus a few global sink tokens. O(T·W) attention. Enables very long inference with bounded KV cache.

the "lost in the middle" problem persists
Even at 128k context, models attend more reliably to tokens at the start and end. Middle tokens are underweighted — empirically demonstrated across GPT-4, Claude, LLaMA-3. Mitigation: put critical information at boundaries, or use retrieval (RAG) instead of stuffing everything into context.

05 MoE — modern implementations LLM

Sparse MoE layer (Qwen3, Mixtral, DeepSeek-V3 style) G(x) = TopK(Softmax(Wg · x), k)   ← router: select k of N experts
out = Σᵢ∈TopK Gᵢ(x) · FFNᵢ(x)

Qwen3-235B-A22B: N=128 experts, k=8 active  22B active / 235B total
Qwen3-30B-A3B: N=128 experts, k=8 active  3B active / 30B total

Fine-grained expert design (DeepSeek-V3 innovation)

Instead of N large experts, use N×m smaller "fine-grained" experts with k×m active. More routing choices, better specialization, more uniform load. DeepSeek-V3 uses 256 fine-grained experts with 8 always-on "shared" experts that every token passes through regardless of routing.

Load balancing loss

Auxiliary loss encouraging uniform expert utilization. Without it: routing collapse (1–2 experts dominate). Applied with small weight (α≈0.01) alongside main loss.

Expert parallelism (EP)

Different experts on different GPUs. Each token is routed via all-to-all communication. Requires large batch sizes to fully utilize all GPUs. Combined with TP and PP.

Shared experts

A small number of experts that every token must use (non-routed). Ensures all tokens share a common processing backbone. Prevents routing instability for rare inputs.

Token-choice vs expert-choice

Token-choice: each token picks its experts. Expert-choice: each expert picks its top tokens. Expert-choice ensures perfect load balance but can drop tokens — not suitable for autoregressive generation.

06 State Space Models & Mamba architecture

Attention is O(T²) in memory. State Space Models (SSMs) process sequences in O(T) with constant memory — making them attractive for very long sequences. Mamba (2023–2025) is the leading SSM for language.

SSM — continuous state space h'(t) = Ah(t) + Bx(t) ← hidden state evolution (ODE)
y(t) = Ch(t) ← output projection

Mamba — selective (input-dependent) SSM Δ, B, C = f(x)   ← parameters are now functions of the input (selectivity)
Standard SSMs: A,B,C fixed. Mamba: B,C,Δ depend on token → can "forget" irrelevant context
why Mamba matters for long sequences
A recurrent model keeps a fixed-size state across all prior tokens — O(1) memory, O(T) compute. Mamba's selectivity lets it decide what to keep in state vs forget, similar to an LSTM gate but for much longer sequences. Parallel scan allows training as fast as attention. Weakness: cannot attend to arbitrary past tokens; recall is compressed into a fixed state.
Mamba-2 & Jamba (hybrid)
Mamba-2 simplifies the selective SSM to be faster on modern hardware. Jamba (AI21 Labs) interleaves attention and Mamba layers — getting O(T) for most context with attention layers for tasks requiring precise long-range recall. Hybrid = best of both worlds in practice.

08 Self-supervised vision — the DINO lineage vision

DINO/DINOv2/DINOv3 is the dominant self-supervised vision paradigm. A student ViT learns to match the outputs of a momentum-averaged teacher ViT, using no labels — just augmented views of the same image.

DINO self-distillation objective L = −P_teacher(x') · log P_student(x)
← teacher sees global crop x', student sees local crop x

Teacher params = EMA(student params): θ_T ← λ·θ_T + (1−λ)·θ_S
λ ≈ 0.996–0.9999 — teacher updates slowly, provides stable targets

Centering: subtract running mean of teacher outputs to prevent collapse
Sharpening: low temperature on teacher softmax to make targets peaky 
pytorch — DINO loss (simplified)
def dino_loss(student_out, teacher_out, center, temp_s=0.1, temp_t=0.04):
    # Center + sharpen teacher output (prevents collapse)
    teacher_p = F.softmax((teacher_out - center) / temp_t, dim=-1)
    student_lp = F.log_softmax(student_out / temp_s, dim=-1)
    loss = -(teacher_p * student_lp).sum(dim=-1).mean()
    # Update center with EMA of teacher outputs
    center = center * 0.9 + teacher_out.mean(dim=0) * 0.1
    return loss, center
DINOv2 additions (2023)
Added iBOT (masked image modeling, patch-level distillation), SwAV (cluster assignment), and KoLeo regularization (spread embeddings uniformly) on top of DINO. The combination of global + patch-level objectives gives DINOv2 its excellent dense feature quality.

09 DINOv3 (Meta, Aug 2025) vision

DINOv3 Meta AI Aug 13, 2025
7B-parameter ViT teacher self-distilled on 1.7B curated images (LVD-1689M). State of the art on classification, depth estimation, segmentation — frozen backbone, no fine-tuning. Closes the gap between self-supervised and supervised foundation models.
  • Scale: 7B ViT teacher (vs ~1B in DINOv2). Student models: ViT-S/B/L/H/g. All released.
  • Data curation (LVD-1689M): Embed all images with a DINOv2 encoder, retrieve 4-NN per curated seed image, k-means cluster to avoid duplicates. 1.7B images vs 142M in DINOv2.
  • Gram anchoring: New loss term stabilizing dense feature quality during long training schedules. Computes Gram matrix of patch features (captures style/texture correlations) and anchors it to early-training values, preventing degradation.
  • High-resolution training stage: Final training phase at higher resolution to sharpen dense features for segmentation and depth tasks.
  • Frozen backbone: All downstream tasks use linear probing or a lightweight head on frozen DINOv3 features — no full fine-tuning needed.

Gram anchoring — the key innovation

Gram matrix of patch features G = FᵀF / (H·W)   F: (H·W)×d patch features · G: d×d

Gram anchoring loss: L_gram = ‖G_current − G_anchor‖²F
← G_anchor set from early training · prevents feature drift during long schedules
Fixes the known DINOv2 problem: dense features degrade after too many training steps. 
pytorch — Gram anchoring loss
def gram_matrix(features):
    # features: (B, HW, d) patch tokens from ViT
    B, N, d = features.shape
    G = torch.bmm(features.transpose(1, 2), features) / N  # (B, d, d)
    return G

def gram_anchoring_loss(student_patches, anchor_gram, weight=0.1):
    G_current = gram_matrix(student_patches)
    loss = weight * ((G_current - anchor_gram)**2).mean()
    return loss
why dense features degrade
Long DINO training schedules optimize global (CLS token) objectives. Over time, patch-level features become less discriminative — they collapse toward the CLS representation. Gram anchoring preserves the second-order patch statistics (texture correlations) throughout training, keeping patch features useful for dense tasks like depth and segmentation.

10 SAM3 (Meta, Nov 2025) vision

SAM3 Meta AI Nov 2025
The Segment Anything Model leap from geometric prompting to semantic concept reasoning. SAM1 segmented by clicks/boxes. SAM2 added video via temporal memory. SAM3 adds language: "segment all ripe apples" — open vocabulary, multi-instance, video.
  • Dual encoder-decoder: 450M vision encoder (Perception Encoder backbone) + 300M causal text encoder trained on 5.4B image-text pairs via contrastive learning.
  • Vision-language fusion: Cross-attention aligns visual tokens and text embeddings. Enables text-prompt → mask without geometric click.
  • Multi-instance open-vocabulary: One text prompt segments all instances (e.g. all apples in a scene). SAM1/2 required per-object prompts.
  • Exemplar-based prompting: Provide an image crop of the concept → segment all matching instances. Few-shot visual grounding.
  • Joint training objectives: Segmentation mask loss + contrastive alignment loss + semantic grounding consistency + temporal memory (inherited from SAM2).
SAM1 → SAM2 → SAM3 progression
SAM1 (2023): Large ViT encoder + prompt encoder + mask decoder. Points/boxes/masks → image segmentation.
SAM2 (2024): Adds streaming temporal memory (key-value store across frames). Propagates masks across video.
SAM3 (2025): Adds text encoder + contrastive alignment. Geometric → semantic. One text prompt → all instances → all frames. 
pytorch — SAM3 style inference (pseudocode)
# SAM3: text prompt → segment all matching instances
image_tokens = vision_encoder(image)           # (HW, d_vis)
text_tokens  = text_encoder("ripe apples")      # (T, d_text)

# Cross-attention: text queries image

    query=text_tokens,                         # text as query
    key=image_tokens, value=image_tokens       # image as key/value
)

# Concept-conditioned mask decoder (multi-instance)

11 ViT registers vision

A 2023 paper from Meta (used in DINOv2+ and DINOv3) that fixes a subtle artifact in ViT attention maps: certain background patches develop anomalously high attention norms, polluting dense features.

the artifact — and the fix
In large ViTs, some uninformative background patch tokens develop very high-norm representations and attract disproportionate attention. The fix: add a small number of learnable register tokens (like extra [CLS] tokens) that "absorb" this global information, freeing patch tokens to carry purely local features. Result: cleaner, smoother attention maps. Better performance on dense tasks. 
pytorch — registers in ViT
class ViTWithRegisters(nn.Module):
    def __init__(self, d_model, n_registers=4):
        super().__init__()
        self.cls_token  = nn.Parameter(torch.randn(1, 1, d_model))
        self.registers  = nn.Parameter(torch.randn(1, n_registers, d_model))

    def forward(self, patch_tokens):   # (B, N, d)
        B = patch_tokens.shape[0]
        cls  = self.cls_token.expand(B, -1, -1)
        regs = self.registers.expand(B, -1, -1)
        # Prepend CLS + registers to patch sequence
        tokens = torch.cat([cls, regs, patch_tokens], dim=1)
        tokens = self.transformer(tokens)
        # Discard registers at output; use only CLS + patch tokens
        return tokens[:, 0], tokens[:, 1 + len(regs[0]):]

12 CLIP & contrastive alignment multimodal

Contrastive image-text pretraining is the foundation for all multimodal models: CLIP's image encoder is the starting point for SAM3's vision backbone, LLaVA's visual projector, and Qwen3-VL's vision encoder.

CLIP InfoNCE loss (symmetric) L = −(1/N) Σᵢ [ log(exp(simᵢᵢ/τ) / Σⱼ exp(simᵢⱼ/τ)) ← image→text
             + log(exp(simᵢᵢ/τ) / Σⱼ exp(simⱼᵢ/τ)) ] ← text→image

sim(i,j) = (v̂ᵢ · ẑⱼ)   ← cosine similarity, L2-normalized embeddings
τ         = learned temperature scalar (typically 0.07)

13 Vision-Language Model architecture multimodal

VLMs (LLaVA, Qwen-VL, InternVL) share a common three-part structure: a frozen vision encoder, a learned projector, and a language model decoder. The projector bridges the two representation spaces.

VLM forward pass Image → ViT encoder → patch tokens (H×W, d_vis)
      → Projector (MLP or Q-Former) → visual tokens (N', d_model)
Text → Tokenizer → token ids → token embeddings (T, d_model)
      → Concat [visual tokens | text tokens] → LLM decoder → next token
Loss computed on text tokens only; visual tokens are never predicted

DeepStack — Qwen3-VL's multi-level vision fusion

beyond single-layer visual features
Standard VLMs extract features from only the last ViT layer. DeepStack (Qwen3-VL, 2025) injects visual tokens from multiple intermediate ViT layers into early LLM transformer layers via cross-attention. Low layers carry texture/edges; high layers carry semantics. The LLM can query different levels of visual detail as needed. This is similar to FPN (Feature Pyramid Network) in detection — multi-scale feature fusion.

Dynamic resolution & native-resolution encoding

Standard VLMs resize images to a fixed resolution (e.g., 224×224), destroying detail. Qwen3-VL processes images at their native resolution using dynamic patch allocation: the image is divided into patches, and the number of visual tokens scales with image area. A 4K image gets ~4× more tokens than a thumbnail — detail is preserved.

14 Qwen3 family (Alibaba, 2025) LLM

Qwen3 Alibaba / Qwen Team May 2025
Dense (0.6B–32B) + sparse MoE (30B-A3B, 235B-A22B). Trained on 36T tokens across 119 languages. Flagship innovation: a single model that can switch between "thinking" (extended CoT) and "non-thinking" (fast response) modes by adjusting a budget parameter — no separate model deployments needed.
  • Hybrid thinking: Budget token parameter scales reasoning depth. High budget → think more → better on hard math. Low budget → skip CoT → fast chat. One model serves both use cases.
  • Architecture: GQA (grouped-query attention), SwiGLU FFN, RMSNorm, RoPE. Standard modern stack.
  • 151,669 vocab: Byte-level BPE. Better multilingual tokenization efficiency vs GPT-4's cl100k.
  • Multimodal extensions: Qwen3-VL (256k context, DeepStack, native resolution, interleaved video), Qwen3-Omni (Thinker-Talker MoE for speech), Qwen3-TTS (streaming speech synthesis).
MRoPE — multimodal rotary position encoding
Standard RoPE encodes 1D position. For video+text, Qwen3-VL uses MRoPE: three separate rotary encoding streams for temporal position, image height, and image width. A video frame token has a (t, h, w) triplet; a text token has (pos, 0, 0). This allows the model to reason about "token at time t, row h, column w" while sharing a unified position encoding space with text.

15 Test-time compute scaling 🔥 hot

The 2024–2025 insight that changed post-training: more tokens at inference → better answers on hard problems. Scaling training compute has diminishing returns; scaling inference compute (thinking tokens) is a new axis.

Chain-of-thought (CoT)

Generate reasoning steps in text before the answer. Each step is a token — the model can use it as a scratch-pad. "Let's think step by step" → 10–30% accuracy gain on math.

Extended thinking / thinking tokens

<think>…</think> tokens before the final answer. The model is trained to use these for exploration, self-correction, and verification. o1/R1 style.

Best-of-N / majority voting

Sample N independent outputs; take the majority answer (for math) or use a verifier to pick the best. N=64 samples often beats a model 10× larger at N=1.

Tree search (MCTS)

Monte Carlo Tree Search over reasoning steps. Expand promising branches, backtrack from bad ones. Highest accuracy at high compute budgets. PRM (Process Reward Model) scores each step.

test-time scaling — compute trade-offs N=1, T tokens: baseline
N=K samples, T tokens each: K× compute, ~√K accuracy gain (diminishing)
N=1, K×T thinking tokens: K× compute, often better than K samples for structured reasoning
Key insight: quality scales more predictably with thinking tokens than with samples

16 GRPO & RLVR (DeepSeek-R1, 2025) 🔥 hot

Group Relative Policy Optimization is the RL algorithm behind DeepSeek-R1's reasoning. It eliminates the critic (value function) of PPO, reducing memory by ~50%, while achieving comparable or better results for reasoning tasks.

GRPO objective For prompt q, sample G outputs {o₁,…,oG} from old policy π_θ_old
Compute rewards {r₁,…,rG} (rule-based: correct=1, wrong=0)

Advantage: Â_i = (rᵢ − mean(r)) / std(r) ← group-relative baseline

L_GRPO = −E[ min(ρᵢ·Âᵢ, clip(ρᵢ, 1±ε)·Âᵢ) ] + β·KL(π_θ ‖ π_ref)
ρᵢ = π_θ(oᵢ|q) / π_θ_old(oᵢ|q) ← probability ratio (same as PPO)
GRPO vs PPO — the key difference
PPO needs a critic network (same size as the policy) to estimate V(s) as the baseline. GRPO replaces V(s) with the group mean reward — no separate network needed. For reasoning tasks (where the only signal is "correct/incorrect"), group mean is a better baseline anyway. Result: half the memory, same or better reasoning performance.

RLVR — Reinforcement Learning from Verifiable Rewards

why verifiable rewards beat neural reward models
Neural reward models (RLHF) are vulnerable to reward hacking — the policy learns to generate text that scores high on the RM without being actually correct. Verifiable rewards (math correctness via symbolic checker, code via compiler/test cases) can't be hacked — either the answer is right or it isn't. DeepSeek-R1-Zero used only two reward signals: accuracy (is the answer correct?) and format (did the model use <think> tags?). Nothing else — and it learned to reason. 
python — GRPO reward computation
def compute_grpo_advantages(rewards: list[float]) -> list[float]:
    # rewards: list of G scalar rewards for one prompt's outputs
    mean = sum(rewards) / len(rewards)
    std  = (sum((r - mean)**2 for r in rewards) / len(rewards)) ** 0.5
    return [(r - mean) / (std + 1e-8) for r in rewards]

# Verifiable reward: math correctness check
def math_reward(prediction: str, ground_truth: str) -> float:
    return 1.0 if parse_answer(prediction) == ground_truth else 0.0

17 Thinking models — emergent behaviors 🔥 hot

DeepSeek-R1-Zero showed that pure RL (no supervised CoT) causes emergent reasoning behaviors. The model is never told how to reason — it discovers strategies through trial and error.

emergent behaviors from RL training
Increasing response length: The model naturally generates longer chains of thought as training progresses — not because it's rewarded for length, but because more thinking = more correct answers = more reward.

Self-correction / reflection: The model learns to re-evaluate intermediate steps, backtrack, and try alternative approaches — discovering metacognition through reward signals alone.

"Aha moments": The model produces statements like "Wait, let me reconsider..." — a self-improvement strategy that emerged from RL, never explicitly trained.
pitfalls of GRPO — length bias
Vanilla GRPO has a bias toward longer outputs: incorrect responses get pushed to be longer (the model hedges with more tokens). The Dr. GRPO paper (2025) identifies this as a normalization artifact and removes the length and std normalization terms, getting better token efficiency while maintaining reasoning quality. Lesson: every loss function has inductive biases worth auditing.

18 μP — Maximal Update Parametrization training

The standard parametrization (SP) does not transfer hyperparameters across model widths. The optimal learning rate for a 125M model is different from a 13B model — so you'd have to tune for each size. μP (Greg Yang, 2022, now standard in frontier labs) fixes this.

μP — key change to initialization and learning rates Standard: W ~ N(0, 1/fan_in) → lr needs re-tuning as width d changes
μP:      W ~ N(0, 1/fan_in) → lr scales as 1/d for weight matrices

Result: tune lr, momentum, wd on a tiny "proxy" model → transfer to 70B model unchanged
Used by Cerebras, DeepMind (Chinchilla), Microsoft (Phi series), EleutherAI
why this matters in practice
Hyperparameter sweeps at 70B scale cost millions of dollars. μP lets you sweep on a 100M parameter model, then transfer the optimal config to 70B. The key insight: in standard parametrization, features update by O(1) per step regardless of width — too large for wide models. μP scales updates to O(1/√width) so the effective LR stays constant.

19 Data mixing & training curricula training

Domain mixing ratios

Web text vs code vs math vs books vs scientific papers. Code improves reasoning; math improves structured thinking. Ratios are treated as hyperparameters tuned on validation perplexity per domain.

Multi-epoch risks

Training on the same data more than once causes memorization. Chinchilla-style: scale tokens proportionally with parameters. For "overtrained" small models (LLaMA-3 8B on 15T): accept some memorization, gain better inference economics.

Curriculum learning

Train on easier examples first, progressively increase difficulty. For reasoning models: start with short proofs, scale to multi-step problems. Reduces instabilities from hard examples early in training.

Cooldown phase

Final training phase with a small, high-quality curated dataset. LR drops to near-zero. "Knowledge consolidation" — improves downstream benchmark performance without much additional compute.

data quality > data quantity (LIMA principle)
The LIMA paper (2023) showed 1,000 carefully curated SFT examples outperform 50,000 noisy ones. DINOv3's LVD curation and Qwen3's data filtering apply the same principle at pretraining scale: the signal-to-noise ratio of training data is as important as the total token count.

21 Speculative decoding inference

Autoregressive generation is memory-bandwidth-bound: each step loads the entire model from HBM. Speculative decoding amortizes this cost by verifying k draft tokens in a single large-model forward pass.

speculative decoding — algorithm 1. Draft model generates k tokens: [t₁, t₂, …, tₖ]
2. Target model scores all k tokens in one forward pass (parallel)
3. Accept tᵢ with probability min(1, p_target(tᵢ) / p_draft(tᵢ))
4. On first rejection, resample from corrected distribution; stop

Lossless: output distribution identical to pure target model sampling
Speedup: 2–3× typical. Scales with acceptance rate α^k
EAGLE-2 (2025)

Draft model uses a single transformer layer on top of target model's hidden states — no separate model. High acceptance rate because draft sees target's features. ~3× speedup on Llama-3.

Self-speculative (Mamba)

Use shallower layers of the same model as the draft. Works naturally with recurrent architectures. No second model needed — just early exit.

Lookahead decoding

Maintain multiple parallel "branches" without a draft model. Fills a Jacobi iteration matrix in parallel. Works on any model. Lower acceptance but no draft model needed.

Reasoning-aware speculative

Reasoning models repeat patterns (verification steps, arithmetic). N-gram models can draft these cheaply. STAND (2025): 60–65% latency reduction on AIME/GPQA.

23 Serving at scale — vLLM, SGLang inference

Paged attention (vLLM)

KV cache managed like OS virtual memory — fixed-size pages allocated on demand. Eliminates fragmentation. Enables much larger effective batch sizes. Foundation of vLLM.

Continuous batching

Don't wait for all sequences to finish before starting new ones. Insert new requests as slots free. Keeps GPU at near-100% utilization. 2–5× throughput vs static batching.

Prefix caching

If many requests share a common prefix (system prompt), compute it once and cache KV. vLLM and SGLang support this. Critical for agent systems with long system prompts.

Chunked prefill

Process long prompts in chunks to avoid OOM. Mix prefill and decode in the same batch. Better GPU utilization when prompt lengths vary widely.

SGLang — structured generation
SGLang adds a RadixTree KV cache (prefix tree for shared prefixes across requests), constrained decoding (JSON schema enforcement via guided decoding), and speculative decoding — all in a single framework. For agent and tool-use scenarios, it dramatically reduces latency for repeated system prompts and structured output requirements.

24 What to know cold

  • RMSNorm over LayerNorm: no mean subtraction, no β, ~15% faster, same stability
  • SwiGLU: three matrices, SiLU gate, hidden = (2/3)×4d for iso-FLOP. Consistently better than ReLU.
  • RoPE: encodes relative position by rotating Q/K. Basis for all long-context extensions (YaRN, LongRoPE)
  • MoE: active params per token << total params. Load balancing loss prevents routing collapse. Fine-grained + shared experts = DeepSeek-V3/Qwen3 style.
  • Mamba: selective SSM — parameters are input-dependent. O(T) memory. Weakness: fixed-size state.
  • DINOv3: 7B teacher, 1.7B images, Gram anchoring prevents dense feature degradation during long training
  • Gram anchoring: regularizes second-order patch statistics (Gram matrix) to stay close to early-training values
  • ViT registers: extra learnable tokens absorb outlier attention, clean patch features for dense tasks
  • SAM3: geometric prompting (SAM1/2) → semantic concept reasoning. Dual encoder: 450M vision + 300M text. Text prompts → multi-instance segmentation.
  • VLM recipe: frozen CLIP encoder → MLP projector → LLM decoder. DeepStack = multi-layer ViT features injected at different LLM depths.
  • GRPO: PPO without a critic. Group mean reward as baseline. Verifiable rewards (RLVR) prevent reward hacking.
  • Emergent reasoning: chain-of-thought length increase, self-correction, and reflection emerge from RL signals alone — not from supervised CoT demonstrations.
  • Test-time scaling: more thinking tokens (extended CoT) is a separate axis from training scale. Budget parameter in Qwen3 controls this at inference time.
  • μP: tune hyperparameters on a small proxy model, transfer to large model. LR ∝ 1/d for weight matrices.
  • Speculative decoding: lossless 2–3× speedup. Draft k tokens with small model, verify in one large-model pass.
  • AWQ/GPTQ INT4: <1% quality loss vs FP16. AWQ protects high-activation-magnitude weights. FP8 for training on H100/B200.

The meta-lesson of 2024–2025: the architecture matters less than the recipe. RMSNorm + SwiGLU + RoPE + GQA is the standard stack — every frontier model uses it. What separates them is data curation (DINOv3's LVD, Qwen3's 36T), post-training methodology (GRPO/RLVR for reasoning, DPO for alignment), and inference engineering (speculative decoding, paged attention, prefix caching). The next capability jump is as likely to come from a better training curriculum as from a new attention variant.
