ML sits on four mathematical pillars: linear algebra (shapes and transformations), probability (uncertainty), calculus (optimization), and information theory (compression). Understanding the geometry — not just the formulas — is what separates fluent practitioners from formula-appliers.

01 Vectors & norms

A vector in ML is almost always a data point or a set of parameters. Norms measure their magnitude — each norm encodes a different geometry.

common norms \(\|x\|_1 = \sum_i |x_i|\) ← L1 · Manhattan · induces sparsity
\(\|x\|_2 = \sqrt{\sum_i x_i^2}\) ← L2 · Euclidean · default distance
\(\|x\|_\infty = \max_i |x_i|\) ← L∞ · max norm · used in robust optim
\(\|x\|_p = \bigl(\sum_i |x_i|^p\bigr)^{1/p}\) ← general Lp norm
Dot product

xᵀy = Σxᵢyᵢ = ‖x‖₂‖y‖₂ cos θ. Measures alignment. If orthogonal: xᵀy = 0. Foundation of every linear layer.

Cosine similarity

cos(x,y) = xᵀy / (‖x‖‖y‖). Measures direction, not magnitude. Standard metric for embeddings, retrieval.

numpy
import numpy as np

x = np.array([3.0, 4.0])

np.linalg.norm(x)           # L2 = 5.0
np.linalg.norm(x, 1)        # L1 = 7.0
np.linalg.norm(x, np.inf)   # L∞ = 4.0

# Cosine similarity
def cosine(a, b):
    return np.dot(a, b) / (np.linalg.norm(a) * np.linalg.norm(b))

# Unit vector (L2-normalize) — embeddings are often pre-normalized
x_hat = x / np.linalg.norm(x)  # ‖x_hat‖₂ = 1.0

03 Eigendecomposition

An eigenvector of A is a direction that the transformation only scales, not rotates. The eigenvalue is that scale factor.

eigenvalue equation \(Av = \lambda v \;\Rightarrow\; (A - \lambda I)v = 0 \;\Rightarrow\; \det(A - \lambda I) = 0\)

eigendecomposition (square diagonalizable matrix) \(A = Q\Lambda Q^{-1}\)
Q: columns are eigenvectors · Λ: diagonal matrix of eigenvalues

for symmetric matrices (A = Aᵀ) — spectral theorem \(A = Q\Lambda Q^T\) ← Q is orthogonal; eigenvalues are real
Covariance matrices, Hessians, kernel matrices are all symmetric → always diagonalizable
geometric interpretation
Eigenvectors are the natural axes of the transformation. In those directions the matrix just stretches (λ > 1), shrinks (0 < λ < 1), reflects (λ < 0), or collapses (λ = 0). PCA finds the eigenvectors of the covariance matrix — the directions of maximum variance. 
numpy
A = np.array([[3, 1],
              [1, 3]])    # symmetric → real eigenvalues

eigenvalues, eigenvectors = np.linalg.eigh(A)
# eigh: for symmetric/Hermitian — more stable than eig
# eigenvalues sorted ascending: [2.0, 4.0]
# eigenvectors: columns are unit eigenvectors

# Reconstruct A from eigen-decomposition
A_reconstructed = eigenvectors @ np.diag(eigenvalues) @ eigenvectors.T

# Power iteration — finds dominant eigenvector cheaply
def power_iter(A, steps=100):
    v = np.random.randn(A.shape[0])
    for _ in range(steps):
        v = A @ v; v /= np.linalg.norm(v)
    return v                   # approximates leading eigenvector
key facts to know cold ✓ tr(A) = Σλᵢ    det(A) = Πλᵢ
✓ Symmetric A → real eigenvalues, orthogonal eigenvectors
✓ PSD matrix → all λᵢ ≥ 0
✓ Rank = # non-zero eigenvalues
→ Largest |λ| controls how fast Aⁿx grows/shrinks (spectral radius)

04 Singular Value Decomposition (SVD)

SVD generalizes eigendecomposition to any matrix — rectangular included. It is the fundamental matrix factorization: every matrix is a sequence of rotate → scale → rotate.

SVD factorization \(A = U\Sigma V^T\)
A: m×n · U: m×m orthogonal · Σ: m×n diagonal (\(\sigma_1\geq\sigma_2\geq\cdots\geq 0\)) · V: n×n orthogonal

geometric interpretation
Vᵀ rotates input space to align with singular directions.
Σ scales each direction by the singular values (σᵢ = √λᵢ of AᵀA).
U rotates to output space.
The singular values σᵢ measure how much each dimension contributes — rank-k approximation keeps only the top-k.

Truncated SVD — best rank-k approximation

Eckart–Young theorem \(A_k = U_k\Sigma_k V_k^T = \underset{\operatorname{rank}(B)=k}{\operatorname{argmin}}\,\|A - B\|_F\)
No other rank-k matrix is closer to A in Frobenius or spectral norm. 
numpy — SVD & low-rank approx
A = np.random.randn(100, 50)

# Full SVD
U, s, Vt = np.linalg.svd(A, full_matrices=False)
# U: (100,50)  s: (50,)  Vt: (50,50)  — "economy" SVD

# Rank-k approximation
k = 10
A_k = (U[:, :k] * s[:k]) @ Vt[:k, :]
# Reconstruction error
error = np.linalg.norm(A - A_k, 'fro')
# Explained variance ratio
var_explained = (s[:k]**2).sum() / (s**2).sum()

# Pseudo-inverse via SVD (most stable method)
A_pinv = Vt.T @ np.diag(1/s) @ U.T

# Condition number — ratio of largest to smallest singular value
cond = s.max() / s.min()   # large → ill-conditioned (numerically unstable)
SVD in ML — where you see it
PCA (dimensionality reduction) · LSA/LSI (text embeddings) · Collaborative filtering (matrix factorization for recommenders) · Pseudo-inverse (least squares) · Weight matrix analysis (singular value spectrum of neural net layers) · LoRA (low-rank adaptation of LLMs decomposes weight updates as UV)

05 Principal Component Analysis (PCA)

PCA finds the orthogonal directions of maximum variance in data. It is both a dimensionality reduction and a change of basis — projecting data into the subspace that preserves the most information (variance).

PCA derivation 1. Center: \(\tilde{X} = X - \mu\) subtract per-feature mean
2. Covariance: \(C = \tfrac{1}{n}\tilde{X}^T\tilde{X}\) shape (d×d)
3. Eigendecompose: \(C = Q\Lambda Q^T\) or equivalently, SVD of X̃
4. Project: \(Z = \tilde{X}\,Q_{[:,\,:k]}\) shape (n×k) — k principal components

explained variance \(\text{var}(i) = \lambda_i \big/ \textstyle\sum_j \lambda_j\)

numpy — PCA from scratch
def pca(X, k):
    # 1. Center
    mu  = X.mean(axis=0)
    X_c = X - mu

    # 2. SVD of centered data (equivalent to eigen of covariance)
    U, s, Vt = np.linalg.svd(X_c, full_matrices=False)
    components = Vt[:k]              # k principal directions (d each)

    # 3. Project
    Z = X_c @ components.T          # (n×d) @ (d×k) → (n×k)

    # 4. Explained variance
    var_ratio = (s**2) / (s**2).sum()

    return Z, components, var_ratio

# Reconstruct (for autoencoder comparison)
def pca_reconstruct(Z, components, mu):
    return Z @ components + mu      # decode back to original space
PCA vs SVD on raw data vs covariance
Running SVD on centered data X̃ gives identical principal components as eigendecomposing the covariance X̃ᵀX̃/n — but SVD is more numerically stable. Never form the covariance matrix explicitly for high-d data (d ≫ n); use randomized SVD instead.
PCA limitations
Linear only — misses non-linear structure (use t-SNE, UMAP for visualization). Sensitive to scale — always standardize first. Directions of max variance ≠ directions most useful for a downstream task (supervised dimensionality reduction like LDA may be better).

06 Projections & least squares

Projection is the operation underlying linear regression, PCA, and attention mechanisms. The projection of b onto the column space of A is the closest point in that subspace.

orthogonal projection onto col(A) \(P = A(A^TA)^{-1}A^T\) symmetric, idempotent: \(P^2 = P\)
\(\text{proj} = Pb\)

least squares solution (Ax ≈ b) \(x^* = (A^TA)^{-1}A^Tb = A^+b\) normal equations · A⁺ is pseudo-inverse
geometric reading: project b onto col(A), find coordinates in that basis 
numpy
# Least-squares regression (numerically stable)
x_star, residuals, rank, sv = np.linalg.lstsq(A, b, rcond=None)

# Projection matrix onto col(A)
P = A @ np.linalg.inv(A.T @ A) @ A.T  # fine for small A
P = A @ np.linalg.pinv(A)             # numerically preferred

# Verify idempotent: P @ P == P
np.allclose(P @ P, P)   # True

07 Probability basics

fundamental rules \(P(A \cup B) = P(A) + P(B) - P(A \cap B)\) ← inclusion-exclusion
\(P(A \cap B) = P(A)\cdot P(B|A)\) ← product rule / chain rule of prob
\(P(A,B) = P(A|B)P(B) = P(B|A)P(A)\) ← joint = conditional × marginal
\(P(A) = \sum_B P(A,B)\) ← marginalization (sum/integrate out B)
Independence: \(P(A,B) = P(A)P(B)\)
Discrete r.v.

PMF: P(X=x) ≥ 0, ΣP(X=x)=1. E[X] = Σx·P(x). Examples: Bernoulli, Categorical, Poisson.

Continuous r.v.

PDF: p(x) ≥ 0, ∫p(x)dx=1. E[X] = ∫x·p(x)dx. Examples: Gaussian, Uniform, Beta, Gamma.

Gaussian — memorize these properties
Sum of Gaussians is Gaussian. Linear transformation of Gaussian is Gaussian. Marginals and conditionals of a joint Gaussian are Gaussian. Max-entropy distribution for fixed mean and variance. These properties make Gaussians analytically tractable — crucial for Kalman filters, GPs, VAEs.

09 Bayes' theorem

Bayes' theorem \(\displaystyle P(\theta\mid X) = \frac{P(X\mid\theta)\cdot P(\theta)}{P(X)}\)

posterior \(\propto\) likelihood \(\times\) prior

\(P(\theta\mid X)\) ← posterior: belief after seeing data
\(P(X\mid\theta)\) ← likelihood: how probable is data given parameters
\(P(\theta)\) ← prior: belief before seeing data
\(P(X)\) ← evidence (normalizing constant, often intractable)
intuition
Bayes' theorem is just conditioning: update your prior belief by how well the hypothesis explains the data. The evidence P(X) = ∫P(X|θ)P(θ)dθ is the hard part — most modern ML avoids computing it exactly (variational inference, MCMC, normalizing flows).

10 MLE & MAP estimation

maximum likelihood estimation (MLE) \(\theta_{\text{MLE}} = \underset{\theta}{\operatorname{argmax}}\;P(X\mid\theta) = \underset{\theta}{\operatorname{argmax}}\sum_i \log p(x_i\mid\theta)\)
← maximize log-likelihood (log for numerical stability, same optimum)
MSE loss ↔ Gaussian likelihood · BCE loss ↔ Bernoulli likelihood

maximum a posteriori (MAP) \(\theta_{\text{MAP}} = \underset{\theta}{\operatorname{argmax}}\;\log P(X\mid\theta) + \log P(\theta)\)
← MLE + log prior · prior acts as regularizer
Gaussian prior → L2 regularization · Laplace prior → L1 regularization
why cross-entropy = negative log-likelihood
For a Categorical distribution parameterized by softmax outputs ŷ,
−log P(y|ŷ) = −Σ yᵢ log ŷᵢ = cross-entropy.
Minimizing cross-entropy loss is maximum likelihood estimation for classification. 
numpy — Gaussian MLE
# MLE for Gaussian: closed-form
mu_mle  = X.mean(axis=0)
var_mle = X.var(axis=0)           # biased (1/n)
var_unbiased = X.var(axis=0, ddof=1) # unbiased (1/(n-1))

# MAP with Gaussian prior = ridge regression
# (XᵀX + λI)w = Xᵀy  ← λ comes from the prior variance
lambda_ = 0.01
w_map = np.linalg.solve(
    X.T @ X + lambda_ * np.eye(X.shape[1]),
    X.T @ y

11 Expectation, variance & covariance

key identities \(\mathbb{E}[aX+b] = a\,\mathbb{E}[X]+b\) ← linearity of expectation (always holds)
\(\operatorname{Var}[X] = \mathbb{E}[X^2] - (\mathbb{E}[X])^2\) ← computational formula
\(\operatorname{Var}[aX+b] = a^2\operatorname{Var}[X]\)
\(\operatorname{Cov}[X,Y] = \mathbb{E}[XY] - \mathbb{E}[X]\,\mathbb{E}[Y]\)
\(\operatorname{Cov}[X,X] = \operatorname{Var}[X]\)

covariance matrix (d-dimensional random vector) \(\Sigma = \mathbb{E}[(X-\mu)(X-\mu)^T]\) shape d×d · always PSD
Sample: \(C = \tilde{X}^T\tilde{X}\,/(n-1)\) unbiased estimator
law of total expectation (tower property)
E[X] = E[E[X|Y]] — useful when direct computation is hard but conditioning simplifies it. Similarly: Var[X] = E[Var[X|Y]] + Var[E[X|Y]] (law of total variance).

12 Derivatives & gradients

gradient — vector of partial derivatives \(\nabla f(x) = \bigl[\tfrac{\partial f}{\partial x_1},\,\tfrac{\partial f}{\partial x_2},\,\ldots,\,\tfrac{\partial f}{\partial x_n}\bigr]^T\) same shape as x
Points in the direction of steepest ascent

key matrix calculus identities \(\nabla_x(a^Tx) = a\)
\(\nabla_x(x^TAx) = (A+A^T)x = 2Ax\) (if A symmetric)
\(\nabla_w\|Xw-y\|^2 = 2X^T(Xw-y)\) ← MSE gradient
\(\partial(AB)/\partial x\): use chain rule; shapes must be consistent
numerical gradient check — always do this
Verify analytical gradients with finite differences during debugging: (f(x+ε) − f(x−ε)) / (2ε) ≈ df/dx. Use ε≈1e-5. Relative error < 1e-5 is good. If it's large, your backprop has a bug. 
numpy — gradient check
def grad_check(f, x, analytic_grad, eps=1e-5):
    numerical = np.zeros_like(x)
    for i in range(x.size):
        x_plus         = x.copy(); x_plus.flat[i]  += eps
        x_minus        = x.copy(); x_minus.flat[i] -= eps
        numerical.flat[i] = (f(x_plus) - f(x_minus)) / (2 * eps)
    rel_error = np.linalg.norm(numerical - analytic_grad) / (
                np.linalg.norm(numerical) + np.linalg.norm(analytic_grad) + 1e-8)
    return rel_error  # < 1e-5: good. > 1e-3: broken.

13 Chain rule & Jacobians

chain rule — scalar-to-scalar \(\dfrac{d}{dx}f(g(x)) = f'(g(x))\cdot g'(x)\)

chain rule — vector-to-vector (Jacobians) If \(y = f(x)\) with \(x\in\mathbb{R}^n\), \(y\in\mathbb{R}^m\):
\(J = \partial y/\partial x\) shape m×n · entry \(J_{ij} = \partial y_i/\partial x_j\)

\(z = g(f(x)) \;\Rightarrow\; \partial z/\partial x = (\partial z/\partial y)(\partial y/\partial x) = J_g \cdot J_f\)

backprop = vector-Jacobian products (VJPs) Given upstream gradient \(\delta = \partial L/\partial y\) (row vector),
downstream: \(\delta \cdot J = \partial L/\partial x\) never form full Jacobian explicitly
Jacobians of common operations y = Wx + b         ∂y/∂x = W    ∂y/∂W = xᵀ (as outer product)
y = σ(x)          ∂y/∂x = diag(σ(x)·(1−σ(x)))
y = relu(x)       ∂y/∂x = diag(x > 0) (diagonal)
y = softmax(x)    ∂y/∂x = diag(y) − yyᵀ    (but use cross-entropy shortcut)
L = ‖y−t‖²       ∂L/∂y = 2(y−t)

14 Convexity & the loss landscape

convex function definition \(f(\lambda x+(1-\lambda)y) \leq \lambda f(x)+(1-\lambda)f(y)\quad\forall\lambda\in[0,1]\)
← the line segment between any two points lies above the function

equivalent condition (twice differentiable) \(f\text{ convex} \;\Leftrightarrow\; \nabla^2 f(x)\succeq 0\) (Hessian PSD everywhere)

strong convexity \(\nabla^2 f(x)\succeq\mu I\;\;(\mu>0)\) unique minimum, linear convergence guaranteed
Convex losses

MSE, cross-entropy, hinge loss — for linear models, these have a unique global minimum. SGD converges.

Non-convex NNs

Neural nets: non-convex, many local minima and saddle points. But in practice, large overparameterized nets rarely get stuck.

saddle points vs local minima
In high dimensions, local minima are rare — most critical points (∇f=0) are saddle points. The Hessian at a saddle has both positive and negative eigenvalues. SGD noise and momentum help escape saddle points naturally.
second-order optimality conditions Necessary: ∇f(x*) = 0   ← first-order condition (gradient zero)
Local min: ∇²f(x*) ⪰ 0 (PSD)   ← all eigenvalues ≥ 0
Saddle point: ∇²f(x*) indefinite   ← mixed positive and negative eigenvalues

15 Lagrange multipliers & constrained optim

Optimize f(x) subject to equality constraints g(x) = 0. The Lagrangian reformulates this as unconstrained.

Lagrangian \(\mathcal{L}(x,\lambda) = f(x) + \lambda^T g(x)\)
Stationarity: \(\nabla_x\mathcal{L}=0\) and \(\nabla_\lambda\mathcal{L}=g(x)=0\)

KKT conditions (inequality constraints g(x) ≤ 0) \(\nabla f + \sum_i\lambda_i\nabla g_i = 0\) stationarity
\(\lambda_i\geq 0,\;\;g_i(x)\leq 0,\;\;\lambda_i g_i(x)=0\) complementary slackness
where this appears in ML
SVM: maximize margin = constrained QP → Lagrangian → dual problem → kernel trick.
PCA: maximize variance subject to ‖w‖=1 → Lagrangian → eigenproblem.
Sparse coding: minimize reconstruction error s.t. ‖z‖₀ ≤ k.

16 Entropy & KL divergence

Shannon entropy — uncertainty of a distribution \(H(P) = -\sum_x P(x)\log P(x)\) bits if log₂, nats if ln
Uniform → maximum entropy    Deterministic → \(H=0\)

cross-entropy — expected log-loss under P using Q \(H(P,Q) = -\sum_x P(x)\log Q(x)\) ≥ H(P) always
= training objective for classification (P = true labels, Q = model output)

KL divergence — how different Q is from P \(\mathrm{KL}(P\|Q) = \sum_x P(x)\log\dfrac{P(x)}{Q(x)} = H(P,Q)-H(P)\)
\(\mathrm{KL}\geq 0\) always    \(\mathrm{KL}=0\) iff \(P=Q\)    \(\mathrm{KL}(P\|Q)\neq\mathrm{KL}(Q\|P)\)
Not a distance metric (asymmetric). Forward KL — mode-covering.
VAE loss = reconstruction + KL
ELBO = E[log P(x|z)] − KL(Q(z|x) ‖ P(z))
The KL term regularizes the latent space toward the prior (standard Gaussian). Minimizing KL(Q‖P) is "reverse KL" — mode-seeking (favors sharp, zero-forcing distributions). 
numpy
def entropy(p):
    p = p[p > 0]               # avoid log(0)
    return -(p * np.log(p)).sum()

def cross_entropy(p, q):
    q = np.clip(q, 1e-9, None)   # numerical safety
    return -(p * np.log(q)).sum()

def kl_divergence(p, q):       # KL(P ‖ Q)
    mask = p > 0
    return (p[mask] * np.log(p[mask] / np.clip(q[mask], 1e-9, None))).sum()

# Verify: KL(P‖Q) = H(P,Q) - H(P)
cross_entropy(p, q) - entropy(p) == kl_divergence(p, q)  # True

17 Mutual information & information gain

mutual information — shared information between X and Y \(I(X;Y) = H(X)-H(X|Y) = H(Y)-H(Y|X) = \mathrm{KL}(P(X,Y)\,\|\,P(X)P(Y))\)
\(I(X;Y)\geq 0\)    \(I(X;Y)=0\) iff \(X\perp Y\) (independent)

information gain (decision trees) \(\text{IG}(S,A) = H(S) - \sum_v \dfrac{|S_v|}{|S|}H(S_v)\) ← entropy reduction after split on A
Gini impurity: \(1-\sum_i p_i^2\) ← faster approx used in sklearn
where this matters in ML
Feature selection: MI between feature and label — select features with highest MI.
Decision trees: ID3 splits on max information gain; CART uses Gini.
Representation learning: maximize I(encoder output; input) (infoNCE loss, contrastive learning).
ICA: minimize mutual information between components.

18 Key interview intuitions

why does SVD give the best low-rank approximation?
The Eckart–Young theorem: truncating to the top-k singular values minimizes both Frobenius and spectral norm error simultaneously. Geometrically: the singular vectors are the directions that capture the most "energy" (variance) in the data — it's PCA on the matrix itself.
L1 induces sparsity, L2 does not — why?
The L1 ball (a diamond in 2D) has corners on the axes. When the elliptical loss contours touch the constraint set, they most often hit a corner — where one coordinate is zero. The L2 ball is smooth — no corners — so optimal solutions are rarely sparse.
what does the covariance matrix encode?
Each off-diagonal entry Σᵢⱼ = Cov(Xᵢ, Xⱼ) — how much two features move together. Its eigenvectors are the principal components; eigenvalues are the variances in those directions. A diagonal covariance = independent features. Singular covariance = features are linearly dependent.
MLE ↔ loss functions
Gaussian likelihood → MSE loss.
Bernoulli likelihood → Binary cross-entropy.
Categorical likelihood → Cross-entropy.
Laplace likelihood → MAE (L1 loss).
Every standard loss is a negative log-likelihood of some distribution.
why is the gradient orthogonal to level curves?
A level curve is {x : f(x) = c}. Moving along it doesn't change f, so the directional derivative (dot product with gradient) is zero — meaning the gradient is orthogonal to any tangent of the level curve. This is why gradient descent moves perpendicular to contour lines.
condition number & numerical stability
κ(A) = σ_max / σ_min = ‖A‖·‖A⁻¹‖. A large condition number (≫1) means the matrix is nearly singular — small perturbations in input cause large errors in output. Ill-conditioned systems are numerically unstable; features on very different scales produce ill-conditioned (X ᵀX) matrices → always normalize.
