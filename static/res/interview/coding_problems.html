01 Binary Search Search
Classic binary search — find target index
easy

Given a sorted array and a target, return its index or -1. The loop invariant: target is always within [lo, hi] if it exists.

example
nums=[1,3,5,7,9], target=7 → 3
nums=[1,3,5,7,9], target=4 → -1
python
def binary_search(nums: list[int], target: int) -> int:
    lo, hi = 0, len(nums) - 1
    while lo <= hi:
        mid = lo + (hi - lo) // 2  # avoids overflow vs (lo+hi)//2
        if   nums[mid] == target: return mid
        elif nums[mid] <  target: lo = mid + 1
        else:                     hi = mid - 1
    return -1

# Test
assert binary_search([1,3,5,7,9], 7) == 3
assert binary_search([1,3,5,7,9], 4) == -1
TimeO(log n)
SpaceO(1)
key insight
Use lo + (hi - lo) // 2 to prevent integer overflow. Always ask: when does the loop terminate? lo > hi — the invariant is broken, target not in array.
Find leftmost / rightmost insertion point
medium

Two classic variants: find the first position where target could be inserted (lower bound), and the last (upper bound). These underlie bisect_left and bisect_right.

example
nums=[1,3,3,3,5], target=3 → left_bound=1, right_bound=3
python
def left_bound(nums: list[int], target: int) -> int:
    """First index where nums[i] >= target (bisect_left)."""
    lo, hi = 0, len(nums)
    while lo < hi:            # note: strict <, hi starts at len
        mid = (lo + hi) // 2
        if nums[mid] < target: lo = mid + 1
        else:                  hi = mid
    return lo               # lo == hi at exit

def right_bound(nums: list[int], target: int) -> int:
    """First index where nums[i] > target (bisect_right)."""
    lo, hi = 0, len(nums)
    while lo < hi:
        mid = (lo + hi) // 2
        if nums[mid] <= target: lo = mid + 1
        else:                   hi = mid
    return lo - 1          # last index equal to target

# Or use the standard library:
import bisect
bisect.bisect_left([1,3,3,3,5], 3)   # → 1
bisect.bisect_right([1,3,3,3,5], 3)  # → 4
TimeO(log n)
SpaceO(1)
key insight
Change hi = len(nums) (open right end) and use strict <. The loop exits when lo == hi — that position is the answer. In interviews: say "I'll use bisect_left / bisect_right" then implement if asked.
02 Fibonacci — 4 implementations Recursion / DP
Fibonacci: naïve → memo dict → iterative → O(1) space
easy

Every progression teaches something. Know all four — interviewers often ask "can you optimize further?"

python
# 1. Naïve recursion — O(2ⁿ) time, O(n) stack
def fib_naive(n: int) -> int:
    if n <= 1: return n
    return fib_naive(n - 1) + fib_naive(n - 2)


# 2. Memoization with explicit dict — O(n) time, O(n) space
def fib_memo(n: int, memo: dict = {}) -> int:
    if n <= 1: return n
    if n in memo: return memo[n]
    memo[n] = fib_memo(n - 1, memo) + fib_memo(n - 2, memo)
    return memo[n]
# Note: pass memo={} as arg, NOT a mutable default to avoid state leakage
# Cleaner version:
def fib_memo_clean(n: int, memo: dict | None = None) -> int:
    if memo is None: memo = {}
    if n <= 1: return n
    if n in memo: return memo[n]
    memo[n] = fib_memo_clean(n - 1, memo) + fib_memo_clean(n - 2, memo)
    return memo[n]


# 3. Bottom-up tabulation — O(n) time, O(n) space
def fib_dp(n: int) -> int:
    if n <= 1: return n
    dp = [0] * (n + 1)
    dp[1] = 1
    for i in range(2, n + 1):
        dp[i] = dp[i-1] + dp[i-2]
    return dp[n]


# 4. Rolling variables — O(n) time, O(1) space (optimal)
def fib(n: int) -> int:
    a, b = 0, 1
    for _ in range(n):
        a, b = b, a + b
    return a

assert fib(10) == 55
NaïveO(2ⁿ)
Memo / DPO(n)
RollingO(1) space
mutable default trap
Never use def f(memo={}) — the dict persists across calls. Use memo=None and initialize inside. This is a classic Python gotcha that interviewers love to probe.
Climbing stairs — ways to reach top
easy

You can climb 1 or 2 steps at a time. How many distinct ways to climb n stairs? This is Fibonacci in disguise.

example
n=4 → 5 (1+1+1+1, 1+1+2, 1+2+1, 2+1+1, 2+2)
python
def climb_stairs(n: int) -> int:
    if n <= 2: return n
    a, b = 1, 2            # ways(1)=1, ways(2)=2
    for _ in range(3, n + 1):
        a, b = b, a + b    # ways(i) = ways(i-1) + ways(i-2)
    return b

assert climb_stairs(4) == 5

# Generalization: k-step climbing
def climb_k(n: int, k: int = 2) -> int:
    dp = [0] * (n + 1)
    dp[0] = 1
    for i in range(1, n + 1):
        for step in range(1, min(k, i) + 1):
            dp[i] += dp[i - step]
    return dp[n]
TimeO(n)
SpaceO(1)
Longest Common Subsequence
medium

Find the length of the longest subsequence common to two strings. Characters must appear in order but not necessarily contiguous.

example
s="ABCBDAB", t="BDCABA" → 4 ("BCBA")
python
def lcs(s: str, t: str) -> int:
    m, n = len(s), len(t)
    dp = [[0] * (n + 1) for _ in range(m + 1)]
    for i in range(1, m + 1):
        for j in range(1, n + 1):
            if s[i-1] == t[j-1]:
                dp[i][j] = dp[i-1][j-1] + 1
            else:
                dp[i][j] = max(dp[i-1][j], dp[i][j-1])
    return dp[m][n]

assert lcs("ABCBDAB", "BDCABA") == 4
TimeO(mn)
SpaceO(mn)
pattern
2D DP on two strings. Same structure as edit distance — swap the merge rule. Space can be optimized to O(n) by keeping only two rows.
03 BFS — Shortest Path Graphs
BFS on a grid — shortest path from S to E
easy

BFS guarantees shortest path in an unweighted graph. Use a deque — pop from left (O(1)) vs list.pop(0) which is O(n).

python
from collections import deque

def bfs_shortest(graph: dict[str, list[str]], start: str, end: str) -> int:
    """Returns shortest path length in unweighted graph, or -1."""
    if start == end: return 0
    visited = {start}
    queue   = deque([(start, 0)])   # (node, distance)

    while queue:
        node, dist = queue.popleft()
        for neighbour in graph.get(node, []):
            if neighbour == end: return dist + 1
            if neighbour not in visited:
                visited.add(neighbour)
                queue.append((neighbour, dist + 1))
    return -1

# BFS on a 2D grid (4-directional)
def bfs_grid(grid: list[list[int]], sr: int, sc: int, er: int, ec: int) -> int:
    """0=open, 1=wall. Returns steps or -1."""
    R, C = len(grid), len(grid[0])
    visited = {(sr, sc)}
    queue   = deque([(sr, sc, 0)])
    while queue:
        r, c, d = queue.popleft()
        if (r, c) == (er, ec): return d
        for dr, dc in [(-1,0),(1,0),(0,-1),(0,1)]:
            nr, nc = r + dr, c + dc
            if 0 <= nr < R and 0 <= nc < C and grid[nr][nc] == 0 and (nr,nc) not in visited:
                visited.add((nr, nc))
                queue.append((nr, nc, d + 1))
    return -1
TimeO(V + E)
SpaceO(V)
BFS rule
Mark nodes visited when you enqueue them, not when you dequeue. Otherwise duplicates flood the queue. Use deque, not list.
DFS — connected components & cycle detection
medium

DFS explores as deep as possible before backtracking. Count connected components with iterative DFS (avoids recursion limit); detect cycles with color marking.

python
# Iterative DFS — count connected components
def count_components(n: int, edges: list[tuple]) -> int:
    graph: dict[int, list[int]] = {i: [] for i in range(n)}
    for u, v in edges:
        graph[u].append(v); graph[v].append(u)

    visited   = set()
    components = 0

    for node in range(n):
        if node in visited: continue
        components += 1
        stack = [node]
        while stack:
            curr = stack.pop()
            if curr in visited: continue
            visited.add(curr)
            stack.extend(graph[curr])
    return components

# Recursive DFS with cycle detection (directed graph)
# Colors: 0=unvisited, 1=in-stack (grey), 2=done (black)
def has_cycle(graph: dict[int, list[int]]) -> bool:
    color = {node: 0 for node in graph}

    def dfs(u: int) -> bool:
        color[u] = 1                    # mark in-stack
        for v in graph[u]:
            if   color[v] == 1: return True   # back edge → cycle
            elif color[v] == 0 and dfs(v): return True
        color[u] = 2                    # fully explored
        return False

    return any(dfs(u) for u in graph if color[u] == 0)
TimeO(V + E)
SpaceO(V)
iterative vs recursive
Prefer iterative DFS in interviews (avoids Python's 1000 recursion limit). For cycle detection, three-color marking (0/1/2) is cleaner than a boolean visited set — a grey node means "currently on the DFS stack".
Topological sort (Kahn's BFS)
medium

Order nodes in a DAG so every edge u→v has u before v. If a cycle exists, not all nodes can be processed — detect it.

python
from collections import deque

def topo_sort(n: int, edges: list[tuple]) -> list[int]:
    """Kahn's algorithm. Returns [] if cycle detected."""
    in_deg = [0] * n
    adj    = [[] for _ in range(n)]
    for u, v in edges:
        adj[u].append(v)
        in_deg[v] += 1

    queue = deque(i for i in range(n) if in_deg[i] == 0)
    order = []
    while queue:
        u = queue.popleft()
        order.append(u)
        for v in adj[u]:
            in_deg[v] -= 1
            if in_deg[v] == 0:
                queue.append(v)

    return order if len(order) == n else []  # [] = cycle

# Example: course schedule (classic LC 210)
assert topo_sort(4, [(0,1),(1,2),(2,3)]) == [0,1,2,3]
TimeO(V + E)
SpaceO(V)
key insight
Kahn's: repeatedly remove nodes with in-degree 0. If any nodes remain at the end, a cycle exists. DFS-based topo (reverse post-order) also works but Kahn's naturally detects cycles and is easier to explain.
04 Lists & Arrays Arrays
Two Sum — hash map O(n)
easy

Find indices of two numbers that add to target. The hash map stores (value → index) for O(1) complement lookup.

example
nums=[2,7,11,15], target=9 → [0, 1]
python
def two_sum(nums: list[int], target: int) -> list[int]:
    seen = {}   # value → index
    for i, n in enumerate(nums):
        complement = target - n
        if complement in seen:
            return [seen[complement], i]
        seen[n] = i
    return []

assert two_sum([2,7,11,15], 9) == [0, 1]
TimeO(n)
SpaceO(n)
Sliding window maximum
medium

Return the max of each window of size k. A monotonic deque keeps candidates in O(1) per step.

example
nums=[1,3,-1,-3,5,3,6,7], k=3 → [3,3,5,5,6,7]
python
from collections import deque

def sliding_max(nums: list[int], k: int) -> list[int]:
    dq     = deque()   # stores indices; front = max of window
    result = []

    for i, val in enumerate(nums):
        # Remove indices outside window
        while dq and dq[0] < i - k + 1:
            dq.popleft()
        # Remove smaller elements (they can never be max)
        while dq and nums[dq[-1]] < val:
            dq.pop()
        dq.append(i)
        # Window is full
        if i >= k - 1:
            result.append(nums[dq[0]])
    return result

assert sliding_max([1,3,-1,-3,5,3,6,7], 3) == [3,3,5,5,6,7]
TimeO(n)
SpaceO(k)
monotonic deque pattern
The deque stores indices in decreasing value order. New element evicts smaller elements from the back. Expired elements evicted from the front. Front is always the max of the current window.
Merge overlapping intervals
medium

Given a list of intervals, merge all overlapping ones.

example
[[1,3],[2,6],[8,10],[15,18]] → [[1,6],[8,10],[15,18]]
python
def merge_intervals(intervals: list[list[int]]) -> list[list[int]]:
    intervals.sort(key=lambda x: x[0])   # sort by start
    merged = [intervals[0]]
    for start, end in intervals[1:]:
        if start <= merged[-1][1]:          # overlapping
            merged[-1][1] = max(merged[-1][1], end)
        else:
            merged.append([start, end])
    return merged

assert merge_intervals([[1,3],[2,6],[8,10],[15,18]]) == [[1,6],[8,10],[15,18]]
TimeO(n log n)
SpaceO(n)
Maximum subarray sum (Kadane's)
medium

Find the contiguous subarray with the largest sum. Kadane's key insight: extend or restart.

example
[-2,1,-3,4,-1,2,1,-5,4] → 6 (subarray [4,-1,2,1])
python
def max_subarray(nums: list[int]) -> int:
    cur = best = nums[0]
    for n in nums[1:]:
        cur  = max(n, cur + n)   # restart if current sum goes negative
        best = max(best, cur)
    return best

# Variant: also return the subarray indices
def max_subarray_idx(nums: list[int]) -> tuple[int, int, int]:
    cur, best = nums[0], nums[0]
    lo = hi = best_lo = best_hi = 0
    for i, n in enumerate(nums[1:], 1):
        if cur + n < n: lo = i; cur = n
        else:           cur += n
        hi = i
        if cur > best: best, best_lo, best_hi = cur, lo, hi
    return best, best_lo, best_hi

assert max_subarray([-2,1,-3,4,-1,2,1,-5,4]) == 6
TimeO(n)
SpaceO(1)
05 Strings & Substrings Strings
Anagram check & grouped anagrams
easy

Two strings are anagrams if they contain the same characters with the same frequencies. Group all anagrams from a list.

python
from collections import Counter, defaultdict

# Check if two strings are anagrams — O(n)
def is_anagram(s: str, t: str) -> bool:
    return Counter(s) == Counter(t)   # or sorted(s)==sorted(t) but O(n log n)

# Group anagrams — O(n·k) where k = avg word length
def group_anagrams(words: list[str]) -> list[list[str]]:
    groups: dict[tuple, list[str]] = defaultdict(list)
    for w in words:
        key = tuple(sorted(w))          # canonical form
        groups[key].append(w)
    return list(groups.values())

words = ["eat","tea","tan","ate","nat","bat"]
# → [['eat','tea','ate'], ['tan','nat'], ['bat']]
TimeO(n·k log k)
SpaceO(n·k)
Longest substring without repeating characters
medium

Sliding window: expand right, contract left whenever a duplicate enters the window.

example
"abcabcbb" → 3 ("abc")    "pwwkew" → 3 ("wke")
python
def length_of_longest_substr(s: str) -> int:
    last_seen = {}     # char → last index seen
    lo = best = 0
    for hi, ch in enumerate(s):
        if ch in last_seen:
            lo = max(lo, last_seen[ch] + 1)  # jump lo past duplicate
        last_seen[ch] = hi
        best = max(best, hi - lo + 1)
    return best

assert length_of_longest_substr("abcabcbb") == 3
assert length_of_longest_substr("pwwkew")  == 3
TimeO(n)
SpaceO(min(n, |alphabet|))
why max(lo, ...)?
Crucially, lo never moves left. Use max(lo, last_seen[ch]+1) — not just last_seen[ch]+1 — because the duplicate might be before the current window.
Longest common prefix
easy

Find the longest string that is a prefix of all strings in the array. Strategy: use the first string as the candidate prefix and shrink it one character at a time until every other string starts with it.

example
["flower","flow","flight"] → "fl"
["dog","racecar","car"] → ""
python
def longest_common_prefix(strs: list[str]) -> str:
    if not strs: return ""
    prefix = strs[0]
    for s in strs[1:]:
        while not s.startswith(prefix):
            prefix = prefix[:-1]      # trim last character
            if not prefix: return ""
    return prefix

# Pythonic one-liner using zip
def lcp_zip(strs: list[str]) -> str:
    prefix = []
    for chars in zip(*strs):
        if len(set(chars)) > 1: break
        prefix.append(chars[0])
    return "".join(prefix)

assert longest_common_prefix(["flower","flow","flight"]) == "fl"
TimeO(S) — total chars
SpaceO(1)
Longest palindromic substring (expand around centre)
medium

Expand outward from every possible centre (n odd-centres + n-1 even-centres). O(n²) and constant space — better than DP for interviews.

example
"babad" → "bab"   "cbbd" → "bb"
python
def longest_palindrome(s: str) -> str:
    def expand(l: int, r: int) -> str:
        while l >= 0 and r < len(s) and s[l] == s[r]:
            l -= 1; r += 1
        return s[l+1:r]

    best = ""
    for i in range(len(s)):
        odd  = expand(i, i)          # odd-length centre
        even = expand(i, i + 1)      # even-length centre
        best = max(odd, even, best, key=len)
    return best

assert longest_palindrome("babad") in ("bab", "aba")
assert longest_palindrome("cbbd")  == "bb"
TimeO(n²)
SpaceO(1)
Manacher's exists but skip it
Manacher's algorithm is O(n) but rarely asked. Expand-around-centre is O(n²), trivial to implement, and interviewers almost always accept it. Mention Manacher's exists if you want bonus points.
